# TodoList 产品需求文档（PRD）

## 1. 产品概述与目标用户
- **产品定位**：个人日常任务管理工具，强调简约、易用、轻量，无需账号登录即可使用。
- **目标用户**：
  - 需要管理日常任务的个人用户（上班族、学生、自由职业者）。
  - 希望用分类/优先级/截止日期来组织任务的用户。
  - 对隐私敏感，倾向本地数据存储的用户。
- **核心价值主张**：
  - 低门槛：打开即可用，无需注册登录。
  - 高效率：快速创建、搜索、筛选和更新任务。
  - 清晰组织：分类、优先级、截止日期让任务一目了然。
  - 稳定留存：本地存储，刷新不丢失。

## 2. 产品目标与成功指标
- **产品目标**
  1) 在 1 周内交付可用的 MVP，覆盖核心任务管理全流程。
  2) 提供清晰的任务组织与筛选能力，帮助用户快速定位当下最重要的事项。
  3) 保证数据在本地持久化，常规使用下不丢失。
- **成功指标（示例，可在上线后细化）**
  - DAU 留存：第 7 天留存 ≥ 40%。
  - 任务操作成功率：创建/编辑/删除/完成 的成功率 ≥ 99%。
  - 性能：首次加载 < 2s（本地开发/常规网络下），交互响应 < 100ms。

## 3. 范围与不在范围
- **本期范围（MVP + 增强）**：单设备个人使用；任务增删改查；分类/优先级/截止日期；搜索筛选；本地存储持久化；简约风格 UI；基础统计。
- **不在范围**：多端同步、团队协作、推送/提醒、账号体系、云备份（可列为未来方向）。

## 4. 功能列表与优先级
### 4.1 MVP（必须，P0）
1) **任务创建**：标题必填；可选描述/优先级/分类/截止日期。校验标题非空、长度 1-100；描述 0-500。
2) **任务列表展示**：按创建时间倒序；显示标题、优先级、分类、截止日期、完成状态。
3) **标记完成/未完成**：复选框切换；完成态显示删除线/弱化；更新时间戳。
4) **删除任务**：支持删除；需确认对话框防误删。
5) **编辑任务**：在表单中修改任务；保存后更新列表。
6) **本地持久化**：LocalStorage 保存任务与分类；刷新不丢失。
7) **基础样式（简约风格）**：清晰层次、基础色彩体系、hover/focus 反馈、空状态提示。

### 4.2 增强功能（重要，P1）
1) **任务分类/标签**：预设分类（工作/学习/生活），支持自定义单分类；列表展示分类；筛选支持分类。
2) **优先级设置**：高/中/低，颜色区分（红/黄/绿）；默认中；列表与筛选支持。
3) **截止日期**：日期选择；显示距离今天的提示（今天/明天/N 天后/已过期）。过期标红提醒。
4) **搜索与筛选**：
   - 搜索：标题/描述关键词实时过滤。
   - 筛选：状态（全部/进行中/已完成）、优先级、分类。
   - 排序：创建时间、截止日期、优先级。
5) **任务统计（轻量）**：总数、已完成数、进行中数、完成率%。

### 4.3 未来方向（可选，P2）
- 数据导出/备份（JSON/CSV）。
- 主题切换（浅色/深色）。
- 子任务/多标签。
- 云同步、账号体系、协作。
- 提醒/推送。

## 5. 界面与交互设计要求
- **整体风格**：简约、清爽；卡片式任务项；主色蓝 (#3b82f6)、高/中/低优先级红/黄/绿；文字深灰，辅助浅灰。
- **布局**：
  - 典型结构：标题区 → 统计 → 表单 → 搜索/筛选 → 任务列表。
  - 响应式：>768px 双列（表单+过滤 / 列表），≤768px 单列纵向。
- **组件与状态**：
  - 表单：标题输入+可折叠高级选项（描述/优先级/分类/截止日期）；校验提示；提交后清空。
  - 列表项：标题、优先级徽标、分类、截止日期、完成勾选、编辑、删除。
  - 空状态：友好提示 + 引导创建。
  - 反馈：hover/focus/active 状态，删除确认对话框；操作成功的轻量提示。
- **可访问性**：
  - 可聚焦元素的清晰 focus 样式。
  - 合理的语义标签；表单关联 label。
  - 颜色对比度符合基础可读性。
- **动效**：
  - 列表新增淡入，删除淡出；完成状态切换有轻量过渡。

## 6. 技术栈建议
- **框架**：Vue 3 + Vite。
- **状态管理**：Composition API + composables（无需 Pinia/Vuex）。
- **样式**：纯 CSS，使用 CSS 变量管理色板与间距；可选 PostCSS。
- **数据存储**：LocalStorage 封装（统一键名/版本，try-catch 容错）。
- **日期处理**：原生 Date API（需求简单）。
- **构建配置**：Vite 路径别名 @ → src；开发端口自定义。

## 7. 数据模型（核心）
```js
{
  id: string,              // 唯一 ID：时间戳-随机串
  title: string,           // 标题 (1-100)
  description: string,     // 描述 (0-500)
  completed: boolean,      // 是否完成
  priority: 'high'|'medium'|'low', // 默认 medium
  category: string,        // 分类，默认“工作”，支持自定义单分类
  dueDate: string|null,    // ISO 8601，可选
  createdAt: string,       // ISO 创建时间
  updatedAt: string        // ISO 更新时间
}
```
- **存储键名**：
  - `todolist_todos`: 任务数组
  - `todolist_categories`: 分类数组（含预设与自定义）
  - 预留 `todolist_settings`（主题、默认分类/优先级等）

## 8. 非功能性需求（NFR）
- **性能**：
  - 首屏加载 < 2s（本地/常规网络）；交互响应 < 100ms。
  - 任务量 500 条内保持流畅，筛选/搜索即时反馈。
- **可靠性/数据持久化**：
  - LocalStorage 写入失败需容错提示；数据结构版本可预留迁移位。
  - 关键操作（删除）需确认，避免误操作。
- **兼容性**：
  - 现代浏览器最新 2 个大版本（Chrome/Edge/Firefox/Safari）。
  - 移动端 H5 适配常见尺寸。
- **安全与隐私**：
  - 数据仅存本地，不上传；不采集个人身份信息。
  - 防止 XSS：渲染时对用户输入做必要转义（不执行 HTML）。
- **可用性**：
  - 可访问性：键盘可操作，焦点可见；颜色对比度合规。
  - 文案清晰，错误提示友好。
- **可维护性**：
  - 组件单一职责；逻辑封装为 composables；统一常量/样式变量。
  - 代码风格统一，必要处注释。
- **可扩展性**：
  - 预留设置存储与版本；分类支持自定义。
  - 未来可挂载云同步/导出等能力。

## 9. 里程碑与优先级
- **MVP（P0）**：创建/编辑/删除/完成，列表展示，本地存储，基础样式，删除确认。
- **增强版（P1）**：分类、优先级、截止日期、搜索、筛选、排序、轻量统计。
- **未来版（P2）**：导出备份、主题切换、子任务/多标签、云同步/协作、提醒。

## 10. 关键用户流程（概要）
- **创建任务**：填写标题 →（可选）展开高级项 → 校验 → 创建并列表顶部展示 → 表单清空。
- **编辑任务**：点击编辑 → 表单回填 → 修改提交 → 列表刷新显示。
- **删除任务**：点击删除 → 确认对话框 → 确认后淡出移除。
- **搜索/筛选**：输入关键词/切换筛选项 → 列表实时过滤与排序。
- **标记完成**：勾选复选框 → 状态与样式切换 → 更新时间戳。

## 11. 风险与缓解
- LocalStorage 容量与清空风险 → 提示用户导出/备份（未来）；控制冗余数据。
- 任务量大导致性能下降 → 使用 computed 过滤/排序，必要时虚拟列表（>1000 可评估）。
- 跨浏览器差异 → 统一存储封装，添加 try-catch 与默认值。
- 误删 → 强制确认对话框。
